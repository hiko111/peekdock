# PeekDock AI 任务舱 PRD

## 1. 产品一句话

PeekDock 是一个面向一人公司和极小团队创作者的 AI 任务托管硬件，通过 Mac 端桌宠闲置态、小屏端工作态和跨屏转场，把多个 AI 工具的异步运行状态变成低打扰、可一眼扫过、可快速介入的任务舱。

## 2. 目标用户

- 高频使用 Codex、Claude、ChatGPT、即梦、Runway、Figma、Notion、浏览器等工具的独立创作者。
- 同时推进代码、文案、视频、设计、调研等多条 AI 任务线的 OPC 或极小团队成员。
- 对切屏、等待、忘记任务卡点、重新理解上下文非常敏感的人。

## 3. 核心问题

- AI 长任务运行时，用户需要频繁切回不同窗口检查状态，打断心流。
- 多个 AI 工具的运行、失败、等待确认、交付状态分散在不同界面里。
- 任务完成或卡住后，用户经常延迟发现，导致整体交付节奏变慢。
- 长任务完成后，用户需要重新理解上下文，复工成本高。

## 4. 产品目标

- 把分散 AI 工具的异步状态统一显示在外置小屏上。
- 用“Mac 闲置态 -> 小屏工作态 -> Mac 召回态”的跨屏互动表现任务状态迁移。
- 用拟人化 agent 动画降低状态识别成本，同时保留必要的任务信息。
- 在任务完成、失败、需要澄清时提供一键打开、重试、语音输入等快操入口。
- 让用户在主屏保持当前工作，不必频繁主动巡检 AI 工具。

## 5. P0 范围

- Mac 端有一个轻量桌面 agent 层，负责展示闲置态、任务发起转场和任务完成召回。
- ESP32-S3 小屏端展示工作态，至少支持运行中、完成、需要介入、失败四类状态。
- Mac/helper 可向小屏发送任务状态和跨屏转场事件。
- 小屏可横滑或按键切换多 Agent 状态。
- 小屏端可通过按钮或触摸触发最小快操，例如打开结果、重试、唤起对应工具。
- 当前仓库先建立可验证的 Mac/helper 与 ESP32 固件通信协议、转场状态机和小屏动画渲染闭环。
- 黑客松 P0 允许先使用 mock 任务源，演示 Codex、Claude 网页端、即梦三个 agent。
- P0 快操只打开本地 HTML 或图片结果，真实 AI 工具接入作为后续可插拔适配器迭代。

## 6. P1 范围

- Mac 端配置软件：管理 AI 工具、查看任务队列、自定义 agent 形象。
- 更完整的跨屏互动动画和 IP 彩蛋。
- 多 agent 空间切换、时间联动、环境联动。
- 可拆卸随身查看模式。

## 7. 主要功能

- 多 AI 工具接入：Codex、Claude/ChatGPT、即梦等。
- 任务状态展示：闲置、发起、运行中、完成、需要澄清、失败、暂停/排队。
- 跨屏互动：任务发起时 agent 从 Mac 主屏进入小屏；任务完成或需要用户处理时从小屏召回 Mac。
- 多任务横划查看：一屏聚焦一个任务，横向切换其他任务。
- 快操按钮：打开交付物、打开对应应用、重试、语音转录补充指令。
- Agent 拟人化：不同工具对应不同角色和动画状态。
