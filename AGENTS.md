# Agent Instructions

This repository follows the `vibe-coding-standard-workflow` process.

## Working Rules

- Keep product decisions and implementation context in `memory-bank/`.
- Read `PROJECT_CONTEXT.md` before firmware work.
- Do not start implementation work before reading the current design and architecture memory.
- Execute the implementation plan one step at a time.
- After each verified milestone, update `memory-bank/progress.md`.
- After changing important files or module responsibilities, update `memory-bank/architecture.md`.
- Keep `memory-bank/implementation-plan.md` code-free.

## 重要提示

- 写任何代码前必须完整阅读 memory-bank/@architecture.md
- 写任何代码前必须完整阅读 memory-bank/@design-document.md
- 每完成一个重大功能或里程碑后，必须更新 memory-bank/@architecture.md
